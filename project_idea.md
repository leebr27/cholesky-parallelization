## Project Idea (LAPACK): Parallel Cholesky Factorization (DPOTRF)

DPOTRF computes the Cholesky factorization A = LLᵀ of a symmetric positive-definite N×N matrix. This is a core LAPACK routine used in least-squares problems, Kalman filters, and sampling from multivariate Gaussians. A serial implementation walks down the diagonal, computing each column of L with a square root and a sequence of dot products and divisions—straightforward to code. For OpenMP, the trailing matrix update at each step can be parallelized across threads. For MPI, a block-cyclic 2D distribution allows the panel and update phases to be distributed, with broadcasts of the current column panel. For CUDA, the trailing symmetric rank-k update (DSYRK) at each step maps naturally to GPU kernels, while the smaller panel factorization can run on the CPU. Unlike LU, Cholesky requires no pivoting, which simplifies communication and avoids the load-imbalance issues of pivot selection, while still offering substantial parallelism in the update phase.
